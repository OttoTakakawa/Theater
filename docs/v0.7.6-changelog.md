# v0.7.6 更新日志

## 新功能

### 标签池分组折叠
- 标签池按分类分组显示，每组可独立折叠/展开
- 全折叠/全展开按钮
- 标签组排序：按分类顺序 / 按标签数量 / 按作品数量 / 按名称
- 默认按标签数量排序
- 每个标签组右上角"+"按钮，可直接在该分类下新建标签（简化流程，跳过 TagCreateDialog）

### 标签池布局优化
- 标签池从 UniformGrid 改为 WrapPanel + 固定宽度 260px，避免同行同高问题
- 搜索框、添加按钮、排序、全折叠合并到同一行，节省纵向空间
- 标签组内 ScrollViewer 隐藏滚动条但保留滚轮滚动

### 筛选栏折叠
- 新增"收起排序"按钮，可隐藏搜索框、排序等筛选控件
- "收起排序"和"收起标签筛选"按钮并排显示

### 退出确认对话框
- 关闭窗口时弹出确认对话框，显示本次会话统计：
  - 阅读了多少本漫画
  - 修改了多少本漫画信息
  - 新增了哪些 Tag（如有）
- "查看日志"按钮可打开日志面板而不退出
- "是/否"按钮确认或取消退出

### 设置：双击选中 Tag
- 设置中新增"双击选中 Tag"选项
- 勾选后需双击标签才触发筛选，单击无响应

## 修复

### 标签数据丢失
- "+"按钮添加标签时使用 `fire-and-forget` 异步保存，切换书籍可能丢失数据 → 改为 `await` 等待保存完成

### 重定位（Relocate）后数据丢失
- 根本原因：BookId 由 FolderPath 的 SHA256 哈希生成，重定位后路径变了但 ID 没更新
- 新增 `RelocateBook` 方法：在事务中删除旧 ID 记录，用新 ID 插入完整数据
- 扫描阶段增加对"路径不在根目录下但路径存在"的已保存书籍的处理，避免丢失

### 重定位闪退
- `Relocate_Click` 是 `async void` 且无 try-catch，任何未捕获异常导致闪退 → 加 try-catch 保护

### 加载进度抖动
- 资料库初始化时文件夹名长度不一导致进度面板宽度抖动 → 截断到 25 字符

### ReaderWindow 双重 BOM
- ReaderWindow.xaml 文件开头有两个 EF BB BF，导致 XAML 编译失败 → 用 UTF8NoBom 重写
