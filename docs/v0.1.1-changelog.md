## 一、漫画残留清理

### 动机
应用从「漫画管理器」演进为「视频/图集为主、漫画兼容」的双形态产品。大量 UI 文案、tag 类别、导入逻辑仍保留漫画向措辞，对视频用户造成困扰。

### 完成项

| 范围 | 改动 |
|---|---|
| **MainWindow.xaml** | ~20 处静态文案：漫画→作品、阅读→观看、本→部、漫画库→书库、在读→在看 |
| **MainWindow.xaml.cs** | ~51 行 StatusText / MessageBox 文本同步替换；`MapStatusText` 状态映射改为在看/未看 |
| **MangaBook.cs** | `ReadCountText` / `ReadStateText` / `ReadingStatusText` / `ContinueActionText` 全部条件化于 `HasVideo`：视频作品显示"看过/在看/继续观看"，漫画作品保留原"读过/在读/继续阅读" |
| **ImportFolderPreflightDialog** | "单本漫画"→"单部作品"、"多本漫画"→"多部作品"、"包含图片"→"包含视频或图片" |
| **退出摘要** | "本次阅读了" → "本次查看了" |

### 原则
- 视频作品 → 观看/看/部
- 漫画作品 → 阅读/读/本（保留兼容路径，不动阅读器）
- 代码注释中的"漫画"保留（描述准确，不影响用户）

## 四、播放器稳定性

### v0.1.1.5 之前的修复（沿用）
- **MainWindow.xaml.cs** — 窗口关闭用 `Dispatcher.BeginInvoke(new Action(Close))` 替代直接 `Close()`，避免 `InvalidOperationException`
- **pack.ps1** — `IncludeNativeLibrariesForSelfExtract=false`，避免单文件启动时解压原生库的几秒延迟
- **TimelineThumbnailCache.cs** — `--vout=dummy --aout=dummy`（不能用 null，否则 VLC 回退到默认可见窗口）
- **TimelineThumbnailCache.cs** — `--avcodec-hw=none` 软解缩略图，不与主播放器抢 GPU
- **PlayerWindow.xaml.cs** — `--file-caching=1000`；缩略图任务延迟到 `_hasStartedPlayback` 后启动；`_playlistRows.ToList()` 快照避免枚举异常；`catch (ObjectDisposedException)` 防窗口关闭崩溃
- **PlayerWindow.xaml/cs** — 彻底删除图集面板（TabGallery），侧栏只剩 播放列表 / 分段标记 / 设置 三个 tab

### 数据库初始化顺序修复
- **LibraryDatabase.cs** — `CREATE INDEX idx_books_last_opened_at ON books(last_opened_at)` 从主 SQL 块移到 `EnsureColumn` 之后。原位置在新数据库上会因 `last_opened_at` 列不存在而失败，SQLite 中断后续语句，导致 `video_segment_markers` 表未创建。
- **MainWindow.xaml.cs** — `_database.Initialize()` 从异步方法移到构造函数，在 `LoadDetailVideoViewMode()` 之前执行。原顺序在空数据库上查询 `app_settings` 会崩溃。
