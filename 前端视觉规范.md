# 前端视觉规范

本文档约束 WPF 前端的视觉实现方式，目标是避免“局部看着能用、整体不统一”的累积问题。新增或修改 XAML 时优先遵守这里的规则。

## 基础原则

- 不使用系统默认控件外观。按钮、输入框、下拉框、滚动条必须走项目样式或主题资源。
- 不使用胶囊按钮。除非未来明确指定某个场景需要胶囊，否则所有业务按钮都使用矩形圆角。
- 不在业务 XAML 中写死 `CornerRadius="数字"` 或 `CornerRadius="999"`。
- 不新增局部按钮模板。按钮模板统一来自 `MangaReader.Native/Themes/ThemeBase.xaml` 中的 `AppButton`、`AppGhostButton`、`AppDangerButton`。
- 页面局部样式只允许补充尺寸、间距、字号，不重新定义 hover/pressed 模板。

## 圆角 Token

圆角必须引用主题 token：

- `RadiusClip`: 极小裁切、封面内部细节。
- `RadiusProgress`: 进度条轨道。
- `RadiusControl`: 按钮、标签 chip、列表行内的小控件。
- `RadiusPanel`: 卡片、列表项、页面分区容器。
- `RadiusField`: 输入框、筛选框、下拉框。
- `RadiusTag`: 色块预览、强调型标签预览。
- `RadiusComboPopup`: 下拉弹层。
- `RadiusDialog`: 弹窗主体。
- `RadiusDialogShadow`: 弹窗阴影壳。
- `RadiusScrollTrack` / `RadiusScrollThumb`: 滚动条。

禁止重新引入 `RadiusPill`。

## 按钮规格

- 普通主按钮：使用默认 `Button` 或 `AppButton`。
- 次级按钮：使用 `GhostButton` / `AppGhostButton`。
- 危险按钮：使用 `DangerButton` / `AppDangerButton`。
- 页面 header 中的管理动作入口使用 `HeaderActionButton`，例如 `新建候选标签`、`新增作者`。
- 小型工具按钮使用 `TinyButton`。
- 批量管理区的小按钮使用 `BatchActionButton`，不得使用 `Pill` 语义命名。

## 弹窗规格

- 弹窗按钮统一使用 `AppDialogButton`、`AppDialogGhostButton`、`AppDialogCloseButton`。
- 弹窗输入框统一使用 `AppDialogTextBox`。
- 弹窗标题、说明、标签分别使用 `AppDialogTitleText`、`AppDialogSubtitleText`、`AppDialogLabelText`。
- 弹窗列表和统计卡优先使用 `AppDialogPanelBorder`、`AppDialogListItemBorder`。
- 禁止在单个弹窗 XAML 中复制 `DialogButton`、`DialogGhostButton`、`DialogTextBox` 这类局部样式；尺寸、间距可以局部调整，控件模板不允许局部重写。
- 长标题必须允许换行或截断，不允许挤压关闭按钮和右侧操作区。

## 容器层级

- 主内容卡片使用 `RadiusPanel`。
- 页面 header、统计卡、列表外框的半径必须一致，避免 9/10/11/12 混用。
- 二级按钮区域宽度必须低于主导航区域宽度，避免视觉层级反转。
- 不为每个小块再套一层大外壳；只有需要表达明确分区时才使用容器。

## 色彩层级

- 使用主题 Brush，不直接写相近的灰色硬编码。
- 分区标题底色深度应高于其子项选中态。示例：分区标题为层级 2，子项选中态为层级 1。
- 不用纯黑/纯白制造层级，优先使用 `Brush.Surface`、`Brush.SurfaceMuted`、`Brush.BorderSubtle`、`Brush.TextMuted`。

## 阅读器规格

- 阅读器 hover / pressed 状态必须由显式模板控制，不允许透明 `ToggleButton`、`RepeatButton` 等命中层漏出 WPF 默认浅蓝矩形。
- 阅读器下拉控件的展开命中层必须使用纯透明模板，只负责点击，不负责绘制状态。
- 阅读器适应高度的语义是：当前单页，或双页模式下的双页展开整体，被约束进固定阅读视口高度。
- 阅读器适应宽度的语义是：当前单页，或双页模式下的双页展开整体，被约束进固定阅读视口宽度。
- 阅读器适配计算必须以 `PixelWidth` / `PixelHeight` 为基准，不使用受图片 DPI 元数据影响的 WPF `Width` / `Height` 作为缩放基准。
- 阅读器适配可用区域必须扣除 `ScrollViewer.Padding`、图片边距和双页间距，避免适高/适宽后产生细微溢出。
- 阅读器图片承载层必须保持固定视口舞台语义，不能随单张图片宽高比收缩，否则扫图尺寸不一时会造成翻页横向抖动。
- 阅读器竖向单页与双页必须使用固定页槽承接扫图尺寸差异，图片在页槽内 `Uniform` 等比居中；双页模式是两个同规格页槽并排，不允许直接拉伸单页来统一宽度。
- 翻页加载是异步流程，任何延迟回来的旧页面请求都不得重新应用适宽/适高，避免快速滚轮翻页时缩放基准被旧页污染。
- 快速翻页不得在热路径中逐页同步写日志或写 SQLite；阅读进度应做防抖合并，退出阅读器时再强制保存一次。
- 窗口尺寸变化、全屏切换等连续布局事件必须防抖后再应用适配，不允许每个 `SizeChanged` 事件都立即重算缩放。

## 新增 UI 的检查清单

- `rg -n 'CornerRadius="[0-9]|CornerRadius="999|RadiusPill' MangaReader.Native --glob '*.xaml'` 应无命中。
- 新按钮是否复用了现有按钮样式。
- 新容器是否使用 `RadiusPanel` 或明确的弹窗 token。
- 新输入/筛选控件是否使用 `RadiusField`。
- 新弹窗是否复用了 `AppDialog*` 系列样式。
- 阅读器新增命中层是否隐藏了默认 hover/pressed 绘制。
- 是否出现了同级控件宽高不一致。
- 是否同步更新 `漫画阅读器开发文档.md`。
